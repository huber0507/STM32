【*】程序简介

-工程名称：RCC——使用HSE/HSI配置时钟
-实验平台: 野火STM32 霸道 开发板 

【 ！】功能简介：
使用HSE/HSI配置系统时钟，能观察到LED灯变化速率及MCO引脚输出的不同


【 ！】实验操作：

编译下载本程序到开发板并复位，观察LED灯变化速率，可以看到，设置不同的系统时钟，
LED闪烁的快慢不一样，更精确的数据可以使用示波器测量PA8(MCO引脚)查看波形。

/*********************************************************************************************/

【*】 引脚分配

MCO输出引脚：

	MCO输出引脚<--->PA8



/*********************************************************************************************/

【*】程序描述：

< bsp_clkconfig.c >

- 使用HSE配置系统时钟(形参pllmul为PLL的倍频因子)

	1.初始化复位与时钟控制（RCC）
	2.使能HSE，等待它变稳定，不稳定则开启HSE失败，或HSE发生故障了
	3.HSE稳定后，使能FLASH预存取缓冲区，设置SYSCLK周期与内存访问时间比例
	4.设置AHB、APB2、APB1的预分频因子
	5.设置PLL的时钟来源和PLL的倍频因子
	6.开启PLL，并等待PLL稳定
	7.把PLLCLK切换为系统时钟SYSCLK
	8.确定系统时钟切换成功

- 使用HSI配置系统时钟(形数pllmul为PLL的倍频因子)

	1.初始化复位与时钟控制（RCC）
	2.使能HSI，等待它变稳定，不稳定则开启HSI失败，或HSI发生故障了
	3.HSI稳定后，使能FLASH预存取缓冲区，设置SYSCLK周期与内存访问时间比例
	4.设置AHB、APB2、APB1的预分频因子
	5.设置PLL的时钟来源和PLL的倍频因子
	6.开启PLL，并等待PLL稳定
	7.把PLLCLK切换为系统时钟SYSCLK
	8.确定系统时钟切换成功


	* 其中，设置HSE和设置HSI是相同的流程，它们的原理是一样的
	* 只是使用HSI时最大的系统时钟就只能是64MHz，而使用HSE能超频到128MHz（慎用超频）



< bsp_mcooutput.c >

- 初始化MCO引脚PA8

	1.引用结构体
	2.开启GPIO对应的时钟
	3.设置引脚、模式、速度
	4.初始化GPIO（根据设定参数配置 GPIO）



< main.c  >

	1.调用HSI配置系统时钟函数，此时系统时钟的来源为HSI
	1.调用初始化MCO引脚函数，然后设置MCO引脚输出系统时钟
	2.调用初始化LED函数，设置计时数目和对应的LED亮、灭情况



PS：在配套的main函数中，HSE配置系统时钟函数被注释掉了，要想观察到HSE对应的情况，需要把函数前的//删掉，
       然后注释掉HSI_SetSysClock(RCC_PLLMul_16)即可。
